How to Use

STEP 1

Pick services to protect

Check the daemons running on your server — sshd is the minimum for any VPS. Add the Nginx or Apache jails if you serve websites, and postfix/vsftpd for mail or FTP.

STEP 2

Tune the ban policy

Set maxretry (failed attempts allowed), bantime (how long an IP stays banned), and findtime (the counting window). Defaults of 3 / 1h / 10m work well for most servers.

STEP 3

Whitelist your own IP

Add your home or office IP to ignoreip so fail2ban can never lock you out — especially important if you choose a permanent ban time.

STEP 4

Install and apply

Copy or download the file to /etc/fail2ban/jail.local, then run the generated commands: install fail2ban, restart the service, and verify with fail2ban-client status.

사용 방법

설치 후 적용

생성된 파일을 /etc/fail2ban/jail.local에 저장하고 명령어를 실행하세요: fail2ban 설치, 서비스 재시작, fail2ban-client status로 확인.

For most VPS setups, a bantime of 1 hour (1h) is a solid default — long enough to stop brute-force scripts, short enough to forgive a mistyped password. High-value servers often use 24h or 1w. A permanent ban (bantime = -1) is aggressive: use it only together with a correct ignoreip so you never lock yourself out.
maxretry is the number of failed attempts allowed within the findtime window before fail2ban bans the IP. maxretry = 3 with findtime = 10m means: 3 failed logins within 10 minutes triggers a ban. Lower values are stricter; 3–5 is the common range for SSH.
Add your IP or subnet to the ignoreip line in the [DEFAULT] section of jail.local, e.g. ignoreip = 127.0.0.1/8 ::1 203.0.113.42. Fail2ban never bans addresses in this list. If you have a dynamic home IP, whitelist your ISP's subnet or use a VPN with a fixed exit IP.
Yes. Fail2ban inserts its own ban rules on top of your existing firewall — by default via iptables (or nftables on newer distros). On Ubuntu with ufw enabled, fail2ban works out of the box; you can also set banaction = ufw in jail.local to have bans managed as ufw rules. They complement each other: ufw sets your static policy, fail2ban adds dynamic bans.
Run: sudo fail2ban-client set sshd unbanip 203.0.113.42 — replace sshd with the jail name and use the real IP. To list currently banned IPs, run sudo fail2ban-client status sshd. On fail2ban 0.10+ you can also unban an IP from every jail at once with sudo fail2ban-client unban 203.0.113.42.
